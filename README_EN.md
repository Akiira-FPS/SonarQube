## SonarQube – Web Interface

Front-end application built with **Vue 3** and **Vite**.  
It provides a web interface to visualize and manage information coming from SonarQube (code quality, projects, metrics, etc.).

> Note: adjust the functional description above if needed to match your exact use case.

---

### 1. Prerequisites

- **Node.js**: LTS version recommended (≥ 18.x)
- **npm** (or **pnpm** / **yarn**, if you prefer)
- Access to a **SonarQube** instance (local or remote), if the app consumes a SonarQube API

Check installed versions:

```sh
node -v
npm -v
```

---

### 2. Project setup (local)

1. **Clone the repository**

```sh
git clone <REPOSITORY_URL>
cd SonarQube
```

2. **Install dependencies**

```sh
npm install
```

3. **Configure `.env`**

An `.env` file is located at the root of the project (or you can create it if it does not exist).  
Example of typical variables (adapt to your project):

```env
VITE_SONAR_TOKEN=SONAR_TOKEN
VITE_SONAR_URL=SONAR_URL
```

> Never commit secrets (tokens, passwords, etc.) to the Git repository.

---

### 3. Local run for developers

To start the development server with hot-reload:

```sh
npm run dev
```

By default, Vite serves the app at `http://localhost:5173` (or another free port).  
Open this URL in your browser.

#### Useful commands for development

- **Start the dev server**:

```sh
npm run dev
```

- **Run lint / code quality checks**:

```sh
npm run lint
```

- **Local production build**:

```sh
npm run build
```

---

### 4. Production build

To generate an optimized production bundle:

```sh
npm run build
```

The generated static files are located in the `dist/` folder.  
This folder contains everything needed to be served by a web server (Nginx, Apache, etc.).

To locally test the production build (optional), you can use for example `serve`:

```sh
npm install -g serve
serve -s dist
```

Then open the URL shown in the terminal.

Summary of the main scripts (check `package.json`):

- `npm run dev`: start the development server.
- `npm run build`: production build.
- `npm run preview` (if present): local preview of the production build.
- `npm run lint`: lint the code with ESLint.
